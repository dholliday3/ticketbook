For tickets, we need a section just for agent notes. That way you can hide it from view by default (bc agents are thorough and verbose) and can also include it in your system prompt or skills that the agent should modify the agent notes section and attach notes there.

Also nice to have an agent debrief. This needs to be an optional setting that can be configured, in the project where you can opt to have the agent provide a concise debrief of how it implemented a ticket. So we'll want a clean agent section that can be hidden by default, is optional, but can store agent research/notes and then debrief. A nice setting would be writing style for debrief, from very concise to lengthy explanation so that based on the users preference, our instructions to the agent will be to respect their wishes when writing the debrief or planning portion. Let's have very concise be the default.
